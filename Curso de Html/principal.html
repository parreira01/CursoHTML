Ferramentas facilitadoras de desenvolvimento

[VOLTAR]
  • VSCode:é uma ferramentas utilizada para codificar usando diversos tipos de programas de linguagem;

  • Live Server: é um programa que pode ser instalado dentro do VSCode para visualizar sua codificação como página local host dentro do navegador. Para abri-lo basta clicar com o botão direito no arquivo principal 'xxx.html' que foi criado e pedir para abrir como live Server;

  • Http-server: faz o mesmo que o Live Server, porém o mesmo é acionado no CLI do terminal após sua instalação.

Inspeção de Elementos

[VOLTAR]

Essa ferramenta chamada ferramenta do desenvolver, ela pode ser aberta clicando com o botão direito na página que está sendo trabalhada ou alguma página da internet e depois em inspecionar elementos ou clique em F12.

Estrutura Básica do HTML

[VOLTAR]
  1. Abra o VSCode e no explorador crie um novo arquivo para codificar sua página;

  2. Criado o arquivo, monte a estrutura do HTML. No caso do VSCode digitando 'html5' será aberto uma estrutura pré definida.

  3. A estrutura de codificação HTML inicia com a tag “.html>” e finaliza com a tag de fechamento “./html>”.

  4. Dentro da estrutura HTML tem-se a tag “.head>” e a ".Body>", onde a tag “.head>” inicia o cabeçalho que possui informações carregadas antes de serem mostradas para o usuário e fecha com “./head>” e a tag “.body>” que é o corpo da página, onde são mostradas as informações para o usuário e que são fechadas com a tag “./body>”.

  5. Ainda, dentro da tag head encontra-se o “.title>” que é o título da página, usado para ser buscado na internet.

TAGS

[VOLTAR]
  • .html> ./html> mostra a página HTML. Possui tag de fechamento;
  • .head> ./head> mostra o cabeçalho da página. Possui tag de fechamento;
  • .body> ./body> mostra o corpo da página. Possui tag de fechamento;
  • .i> ./i> coloca o texto/palavra em itálico. Possui tag de fechamento;
  • .strong> ./strong> coloca o texto/palavra em negrito e dá ênfase a palavra/texto. Possui tag de fechamento;
  • .b> ./b> coloca o texto/palavra em negrito. Possui tag de fechamento;
  • .input > abre um campo para preenchimento e não possui tag de fechamento;
  • .img /> referencia uma imagem e essa tag é aberta e fechada no mesmo campo;

Atributos das TAGS

[VOLTAR]

Atributos dão valor as tags e possuem vários ligados a outros programas como CSS e JS. Alguns exemplos de atributos estão citados abaixo.

  • id: é um atributo de identificação de uma determinada tag e que também é usado no JS.

    • Ex: .strong id = “titulo”>;
  • class: é um atributo voltado para o CSS que define uma classe de comando que serão alterados de mesma forma;

    • Ex: .strong class = “titulo”>;
  • style: é um atributo para estilizar uma determinada tag e que também é usado no CSS.

    • Ex: .strong style= “color:blue;”>;
  • src (search) : é um atributo de busca que define o caminho de um arquivo na internet ou em um arquivo local. ;

    • Ex: .img src=”htttps://www.dio.com/Dio.me”/>
  • type : é um atributo específico da tag .input> e que classifica o campo como: text, number, color e outros.

    • Ex: .input type=”text”>

Listas

[VOLTAR]

Existe dois tipos de listas: a ordenada 'ol' e não-ordenada 'ul'.

  • Lista ordenada (.ol> ./ol>): possui a característica de precisar de uma tag da lista filha .li> para que a mesma funcione. O nome de lista ordenada se dá pelo fato de aparecer com uma numeração ordenada na frente dos itens listados.
    Ex1:
    ol>
    li> cabeça /li>
    li> ombro /li>
    li> pé /li>
    /ol>
    Resultado:
    1. cabeça
    2. ombro
    3. pé
  • Lista não-ordenada (.ul> ./ul>): tem as mesmas características da lista ordenada, sendo necessária uma lista filha, porém não apresenta números na frente dos itens e sim marcadores.
    Ex2:
    ul>
    li> cabeça /li>
    li> ombro /li>
    li> pé /li>
    /ul>
    Resultado:
    • cabeça
    • ombro
    • pé

Links

[VOLTAR]

São formas de navegar ou ligar a página dentro dela mesma ou a outros sites. A tag de ligação é a ancor 'a'.

  • O 'href' é um atributo específico da tag ancor. Ele indica o endereço de destino a ser ligado.
  • Outro atributo é o 'target', ele indica quando a página a ser ligada pelo 'href' vai abrir na mesma aba do site ou em outra aba, antigamente era usado os valores '_top' e '_parent' para fazer retrocompatibilidade, mas hoje estão em desuso. Para abrir a página na mesma aba o valor a ser selecionado é o '_self' e para abrir em outra aba o valor é '_blank'.
    • Ex: .a href=”xxxx”> Nome que vai aparecer na página ./a>
  • O title é outro atributo da tag ancor que faz com que ao se passar o mouse sobre o nome do link será exibido um balão com a informação gerada por esse atributo.
  • Ainda dentro da tag de ligação podemos ligar um palavra/texto a outra palavra/texto dentro da própria página, assim dentro dessa tag utiliza-se o atributo name= “#nome ao qual será lido” e o valor é acompanhado com o símbolo '#' para demonstrar que será referenciado pelo href= “nome citado no atributo name” pelo atributo 'href' acompanhado do valor que é o mesmo nome do valor do atributo name sem o acompanhamento do símbolo '#'.
    • Ex:
    .a name=”#exemplo1”>Exemplo ./a>

    .a href=”exemplo1”>Exemplo ./a>

  1. tag form


    2. São trechos no HTML, em formato de formulário, onde o usuário coloca informações a serem enviadas e processadas por um DB. Essa tag tem abertura e fechamento.


    Ex:

    .form name:"signup" method="post ou get" target="_blank ou _self" autocomplete= "on ou off" onsubmit="alert('Enviei o formulário')action="https://url da página">
    Nome: .input type=”text” name=”name” /> .br>
    Idade: .input type=”number” name=”age” /> .br>
    Senha: .input type=”password” name=”password” /> .br>
    .Button type=”submit”> Enviar ./Button>
    ./form>
    ./body>
    • O atributo 'name' é para nomear a tag 'form';
    • O atributo 'action' envia o formulário para o site mostrado em seu valor, desde que tenha autorização;
    • O atributo 'method' com os valores 'GET' ou 'POST', sendo o 'GET' uma forma menos segura de enviar o formulário, pois os dados aparecem na barra de endereço da URL, inclusive a senha digitada, já o método 'POST' envia os dados de forma encapsulada ao servidor, onde somente o servidor vai poder desincriptar o formulário;
    • O atributo 'target' é para abrir o formulário, sendo que aberto na mesma aba utiliza o valor'_self' ou em outra aba usando o valor '_blank';
    • O atributo 'autocomplete' pode ter o valor 'on' que faz com que os valores dos campos se autocompletam ou 'off' para completar os dados mecanicamente;
    • Outro atributo usado é o 'onsubmit' com o valor =”alert ('Enviei o formulário')” e uma mensagem entre parênteses, esse atributo revela a informação ao usuário quando o botão 'enviar' é clicado. (O 'alert' e sua mensagem já são mais usuais na implementação com javaScript);
    • Entre a abertura e o fechamento do 'form', é dado o NOME do campo, acompanhado da abertura da tag 'input' que é o campo em si para preenchimento dos dados;
    • Na tag 'input' o atributo 'type' especifica qual o tipo de dado poderá ser editado dentro do campo, podendo ser os valores: 'text' para dados em texto, 'number' para dados em números, 'password' para senhas, existem outros valores para o 'type' e podem ser encontrados W3Schools;
    • E o atributo 'name' que serve para nomear a tag 'input'. Outros atributos da tag 'input' estarão no próximo tópico.

    ESTRUTURAÇÃO E FORMATAÇÃO DO HTML


    O objetivo dessa parte do HTML5 é dar maiores condições de acessibilidade a sua página, aprender como adicionar fontes e cores pelo HTML e preparar a estrutura do código para receber formatações do CSS.

    Logo abaixo está algumas tags que ajudarão na estrutura e formatação dos textos

    1. FORMATAÇÃO DE TEXTO

      2. Essa parte do curso será trabalhada a formatação e semântica dos textos inseridos dentro de uma página HTML, assim serão demonstradas como inserir sublinhados, negritos, itálicos, ou seja, formas de deixar o texto mais atrativo para o usuário.

      Existem várias tags, as mais usadas são:

      1. .b> ./b> =
        2. para negrito sem semântica;
      2. .strong> ./strong> =
        3. para negrito com semântica;
      3. .i> ./i> =
        4. coloca as palavras em itálico;
      4. .u> ./u> =
        5. sublinha as palavras;
      5. .sup> ./sup> =
        6. valor elevado a potência; EX: x^2
      6. .sub> ./sub> =
        7. valor sob o número;
        EX: x_2
      7. .mark> ./mark> =
        8. estilo marca texto;
      8. .font> ./font> =
        9. formata a fonte do texto, nela pode ser usados atributos e valores, como: atributo 'color' que muda a cor da fonte, exemplo de valor red, o atributo 'face' que nomeia o estilo da fonte, por exemplo, o valor arial, etc.
        Ex: .font color=”red” face= “Arial”>
        Programe:o seu futuro global agora!

        Observação:nomes de fontes compostas têm que estar entre aspas simples e o mais indicado é usar a fonte através do HTML.

        Ex: 'trebuchet MS'
    2. TAG DIV E SPAN

      3. São tags de estrutura de página web, são delimitadoras de espaço no HTML para serem melhor formatadas pelo CSS.

      Cada campo criado dessa página, como: cabeçalho, menu, conteúdo e rodapé, são delimitados por uma 'div', porém ela é uma tag que não aparece na tela, mas tem por característica ser uma tag de 'display' do tipo 'block', ou seja, ela ocupa toda parte horizontal da página.

      tag 'span' não tem display em 'block', portanto é utilizada para quando não há necessidade de separação de textos.

      Ex:
      DIO.ME com o ponto em 'div':
      .p> DIO .div> . ./div>me ./p>

      DIO

      me

      Já com a tag 'span':
      .p> DIO .span> . ./span>me ./p>

    6. CORES

      7. Em cada navegador, de acordo com a w3schools, existem aproximadamente 140 nomes de cores disponíveis e elas variam de navegador pra navegador.

      As cores podem ser aplicadas através do nome das mesmas ou pelos valores abaixo:

      • RGB: combinação de vermelho, verde e azul (red, green, blue);
        • EX: rgb (150, 28, 230)
      • HEX: combinação de vermelho, verde e azul, mas descrito em hexadecimal;
        • EX: #FF0000 é vermelho, #00FF00 é verde e #0000FF é azul; Suas medidas são realizadas da seguinte forma:
      • HSL: Matiz (HUE), Saturação (Saturation) e Iluminação (lighten).

        A matiz tem os valores onde 0 é vermelho, 120 é verde e 240 é azul;

        EX: As cores vão de 0 a 360, onde de 0 a 119 é vermelho, 120 a 239 é verde e 240 a 360 é azul, a saturação é o segundo campo e é medido em porcentagem e o terceiro campo é o brilho também em porcentagem. hsl(240, 50%, 60%);
      • RGBA: Mesmo que RGB, porém adicionado canal Alpha (transparência);
        EX: rgba (150, 28, 230, 0.6), onde a intensidade de transparência do ,'a' vai de 0 a 1.
      • HSLA: Mesmo que HSL, porém adicionado canal Alpha (transparência);
        EX: hsla(240, 50%, 60%, 0.7), onde a intensidade de transparência do 'a' vai de 0 a 1.

      Observação: caso queira visualizar as cores, basta digitar o formato da cor no buscador do site w3schools para ser gerado uma página de cálculo de cores.

    TRABALHANDO COM MÍDIAS


    1. TAG IMG

      2. Essa tag captura imagens de arquivos da sua máquina local ou de websites.

      O exemplo de código abaixo mostra como fazer a captura da imagem.

      .body style=”background-color: black”>
      .img src=”endereço do site ou caminho do arquivo local da imagem” >
      .img title=”nome aparente ao passar do mouse em cima da imagem” alt=”descrição detalhada da imagem para acessibilidade”>


        Observação:Existem diversas extensões de imagens, como: gif, jpg, png, svg e outras.

      • gif: a imagem tem movimento, porém possui baixa definição na sua expansão e é um arquivo muito pesado;
      • jpg: é uma imagem compactada, opaca, leve, porém de baixa resolução;
      • png: tem como características: animação, fundo transparente e compressão sem perda de qualidade, é um pouco mais pesada do que o formato 'jpg';
      • svg: é um tipo vetorial, não perde resolução, mais direcionado a construção de logos e desenhos, por conta dos pontos vetoriais e por não ser dimensionados por pixels.O formato svg, pode ser utilizado como tag;
        .svg> PataCachorro ./svg>


      • width=”000” é um atributo CSS usado para diminuir a imagem, ele pode ser utilizado na tag img.
      • Os atributos title e alt dentro da tag img servem: o valor do 'title' é inserido para ser visto quando o mouse passar pela imagem e o valor 'alt' para descrição detalhada da imagem por robô para a acessibilidade a pessoas com deficiência.
      • O atributo align alinha uma imagem a um texto.

    2. TAG AUDIO

      3. Exclusiva para áudio e é nativo do navegador, possui semântica enquanto lido por robô.

      Ex:
      .audio autoplay controls> .audio/>
      .source src=”musical.mp3”/>
      Seu navegador não tem suporte para esse arquivo.

      Os atributos 'autoplay' serve para iniciar o áudio automaticamente e o 'controls' mostra os controles para a condução do áudio.

      A tag src (source) é tag filha da tag áudio

    3. TAG VIDEO

      4. Utiliza os recursos nativos do navegador para reproduzir um vídeo.

      Ex:
      .div>
      .video controls>
      .source src=”clipe.mp4”/>
      .source src=”clipe2.webm”/>
      Seu navegador não suporta vídeo
      ./vídeo>
      ./div>

      Observação: As extensões mp4 e webm, são as únicas aceitas na tag video.

    4. TAG TRACK

      5. Track são legendas que são utilizadas na tag vídeo, através de extensões '.vtt' . Ela coloca o inicio da legenda e o final da legenda.

      .div>
      .video controls>
      .source src=”clipe.mp4”/>
      .source src=”clipe2.webm”/>
      .track src=”legenda.vtt” kind= “captions” srclang=”en”/>
      . .track default src=”legenda-pt-br.vtt” kind= “captions” srclang=”pt-br”/>
      Seu navegador não suporta vídeo
      ./vídeo>
      ./div>

      Observação:

      • O atributo 'default' informa que aquela tag vai iniciar primeira, por padrão;
        Ex: track default..
      • O Atributo 'kind' tem diversos valores: o 'caption' serve para mostrar no vídeo a fala, conversa ou letra da musica, o 'descriptions' é lido detalhadamente por robô sobre o que ocorre no vídeo, o 'chapters' sserve para mostrar pedaços da leitura do vídeo determinado pelo desenvolvedor;
        Ex: kind= “captions”
      • O atributo 'srclang' determina o valor da abreviatura de um idioma que será mostrada no ícone de configuração do vídeo.
        Ex: srclang=”pt-br”
      • A busca da compatibilidade da extensão do clipe vai através da seqüência que ela é colocada se 'mp4' ou se 'webm' até a mensagem de não suporte pelo navegador;
      • O arquivo da extensão '.vtt' é um arquivo de texto comum, ele pode ser editado no seguinte formato:
        Ex: 00:00:11.000 --> 00:00:20.300
        Ela foi pro céu...
        Morreu!

    5. TAG IFRAME

      6. O 'iframe' é a evolução do 'enbed', ela serve para incorporar páginas externas ou vídeos externos dentro da sua página.

      O caminho para capturar o vídeo de uma página é: Abra a página/vídeo acesse compartilhar, depois incorporar e após copiar o caminho do 'iframe' que será disponibilizado e em seguida cole o caminho no local destinado ao 'iframe'.

      Ex: .body>
      .iframe src=”caminho do vídeo alocado em site externo"> ./iframe>
      ./body>

    CRIANDO TABELAS


    Tabelas são a formação tabular de colunas e linhas, onde o encontro dessas é denominado célula.

    A estrutura do site, antigamente, era feito no formato de tabelas, porém essa estrutura em um site tinha problemas com a semântica dos textos, portanto os robôs não conseguiriam ler corretamente o trecho do código do site, ou seja, não saberia do que se trataria. A leitura do HTML feita por esse robô é chamado de 'web screping'. Essa é a base para o SEO que faz a leitura do HTML, identificando: títulos, parágrafos, formatações e semânticas para torná-las mais realçadas para o usuário

    Outro problema era o tamanho do arquivo, pois, nas tabelas o arquivo é maior do que em tags com a div e o span, que dependendo do tamanho do site pode fazer com que ele fique pesado e lento pra carregar as informações para o usuário.

    A acessibilidade também ficava prejudicada quando a tabela era utilizada para estruturar a totalidade do site, pois, o leitor de tela para acessibilidade para um deficiente visual, por exemplo, vai se perder semanticamente na hora de fazer a leitura do texto.
